# REAEDP

**Renyi Entropy Analysis and Evaluation Based $(\varepsilon,\delta)$-Differential Privacy Protection Method under the Wiener Kernel Space**

This repository contains the paper LaTeX source and reproducible experiment code. All Python code and experiments live at the **project root**; a single entry point **`main.py`** runs experiments via **`config.json`**.

---

## Directory layout (top level)

| Path | Contents |
|------|----------|
| **`main.py`** | Entry point: run experiments from `config.json` (no need to run each script separately). |
| **`config.json`** | Parameters for each experiment; edit `"run"` to choose which tasks to run. |
| **`reaedp/`** | Core package (entropy, DP mechanisms, Noisy-SGD, Wiener kernel, image noise). |
| **`experiments/`** | Experiment scripts (entropy sensitivity, Wiener figures, privacy test, image noise, etc.). |
| **`paper/`** | Paper source: `main.tex`, `ref.bib`, `figs/` (figures generated by experiments). |
| **`data/`** | Generated outputs: CSV, image noise results (created when running). |
| **`requirements.txt`** | Python dependencies. |

## How to Run

### Run all experiments from config (recommended)

From the **project root** with venv activated:

```bash
cd c:\source\REAEDP
pip install -r requirements.txt
python main.py
```

This runs every task listed in `config.json` under `"run"` (entropy_sensitivity, noisy_sgd, wiener_figures, privacy_test_fig1, image_noise), using the parameters defined for each in `config.json`.

### Run only selected tasks

```bash
python main.py --run entropy_sensitivity image_noise
```

### Use a different config file

```bash
python main.py --config my_config.json
```

### Edit config.json

- **`"run"`**: list of task names to run (e.g. `["entropy_sensitivity", "wiener_figures", "image_noise"]`).
- **Per-task keys** (e.g. `"entropy_sensitivity"`, `"image_noise"`): parameters for that experiment (e.g. `out_csv`, `epsilon`, `input`, `output`). Adjust paths, epsilon/delta, and data paths here so you don’t need to pass command-line flags.

### Image path in config.json (folder or single file)

To add noise to your images in one go, set **`image_noise.input`** in `config.json`:

- **Whole folder (batch):** set `"input"` to the folder path. All images in that folder (`.png`, `.jpg`, `.jpeg`, `.bmp`) will be processed with the same `epsilon`, `delta`, and `mechanism`. Example:

```json
"image_noise": {
  "input": "data/my_images",
  "output": "data/image_noise",
  "epsilon": 1.0,
  "delta": 1e-5,
  "mechanism": "both",
  "out_csv": "data/image_noise_results.csv",
  "seed": 42
}
```

  Put your images in `data/my_images/` (or any folder), then run `python main.py` or `python main.py --run image_noise`. Noisy images and comparison figures are written to `output` (e.g. `data/image_noise/`).

- **Single image:** set `"input"` to the image file path, e.g. `"data/sample.png"`.
- **Demo (no download):** leave `"input": ""` to use the built-in synthetic image.

Paths are relative to the project root when you run `main.py` from the repo root.

### Run individual scripts (optional)

You can still call one experiment at a time from the **project root**:

```bash
python -m experiments.run_entropy_sensitivity --out_csv data/entropy_sensitivity_results.csv
python -m experiments.run_wiener_figures --out_csv data/wiener_curves.csv
python -m experiments.run_image_noise --input path/to/image.png --output data/image_noise
```

## Generating figures for the paper

The **experiments** produce figures that are used in the paper. Run the figure-producing tasks, then compile the LaTeX; the paper includes them from **`paper/figs/`**.

### Which experiment produces which figure

| Experiment | Output figure(s) | Saved to | Use in paper (`paper/main.tex`) |
|------------|------------------|----------|----------------------------------|
| **`privacy_test_fig1`** | Privacy test pass rate vs. k, γ | `paper/figs/fig1.png` | `\includegraphics{figs/fig1.png}` (Fig. 1) |
| **`wiener_figures`** | Original vs private RKHS mean (3 values of ρ) | `paper/figs/fig2.png`, `fig3.png`, `fig4.png` | Fig. 2–4 (Wiener kernel / penalty ρ) |
| **`image_noise`** | Original vs Laplace vs Gaussian (per image) | `data/image_noise/*_compare.png` | Copy one to `paper/figs/` and add a `\includegraphics` if you want it in the paper |
| **`epsilon_utility_curve`** | PSNR and MAE vs ε (Laplace & Gaussian) | `paper/figs/fig5.png` | Fig. 5 (privacy–utility tradeoff) |
| **`entropy_bound_curve`** | Entropy sensitivity bound Δ_H vs n | `paper/figs/fig6.png` | Fig. 6 (Theorem 4) |
| **`metrics_comparison`** | PSNR / MAE / SSIM bar chart (Laplace vs Gaussian) | `paper/figs/fig7.png` | Fig. 7 (metrics comparison) |
| **`csv_entropy_experiment`** | Histogram + entropy on real CSV (e.g. y_amazon-google-large.csv) | `paper/figs/fig_csv_entropy.png` | Real-data experiment (histogram + DP noise) |

### Run only the experiments that draw paper figures

To generate **fig1.png** and **fig2–fig4.png** (no CSV, no image_noise):

```bash
python main.py --run privacy_test_fig1 wiener_figures
```

To also generate image-noise comparison figures (in `data/image_noise/`):

```bash
python main.py --run privacy_test_fig1 wiener_figures image_noise
```

### Tuning figure quality for the paper

- **DPI:** In **`config.json`**, set **`dpi`** under `privacy_test_fig1` or `wiener_figures` (e.g. `"dpi": 300`) for higher-resolution figures for the paper.
- **Parameters:** k, γ, ρ, ε, δ are set in **`config.json`** under the corresponding experiment. Edit and re-run to get new curves/figures.

### Using the figures in the paper

- **fig1–fig4** are already referenced in **`paper/main.tex`** (e.g. `figs/fig1.png` … `figs/fig4.png`). After running the experiments above, they appear in **`paper/figs/`**; compiling the paper will include them.
- To add an **image-noise** comparison in the paper: copy one of the `*_compare.png` from **`data/image_noise/`** to **`paper/figs/`** (e.g. `paper/figs/fig_image_noise.png`), then in `main.tex` add something like:
  ```latex
  \begin{figure}[ht]
    \centering
   \includegraphics[width=0.8\textwidth]{figs/fig_image_noise.png}
    \caption{Original vs Laplace/Gaussian DP noise on an image.}
    \label{fig:imagenoise}
  \end{figure}
  ```

### One-line summary

```bash
# Generate all paper figures (fig1–fig7) into paper/figs/
python main.py --run privacy_test_fig1 wiener_figures epsilon_utility_curve entropy_bound_curve metrics_comparison
```

**根据已下载数据实时生成论文图（含多表熵图）**：先执行 `python data/download_data.py` 下载 CSV 到 `data/`，再运行下面任一方式，会把 **fig1–fig7** 以及 **每张 CSV 的熵+DP 图** 生成到 `paper/figs/`：

```bash
# 方式一：一键生成全部论文图（含 data/ 里多张 CSV 的熵图）
python main.py --run paper_figures

# 方式二：只生成多张 CSV 的熵图（不跑 fig1–fig7）
python main.py --run csv_entropy_all
```

多表熵图由 **`config.json`** 里 **`csv_entropy_all.datasets`** 控制，每项为 `"input"`（如 `data/house-prices-train.csv` 或相对 `data_dir` 的文件名）、`"column"`（数值列名）。缺文件的项会自动跳过。生成的图名为 `fig_csv_entropy_<文件名>.png`，保存在 `paper/figs/`。

Then build the PDF (see below); the paper will use the generated figures.

## Build the paper PDF

You need a LaTeX distribution that provides `pdflatex` and `bibtex`. If you see **“pdflatex 无法识别”** or **“pdflatex is not recognized”**, install LaTeX and add it to your PATH.

### Install LaTeX (Windows)

1. **MiKTeX** (recommended on Windows): https://miktex.org/download  
   - Run the installer and choose “Install for all users” or “Just for me”.  
   - During setup, enable **“Add MiKTeX to PATH”** (or add the `bin` folder to PATH manually, e.g. `C:\Users\<You>\AppData\Local\Programs\MiKTeX\miktex\bin\x64`).  
   - Restart the terminal (or Cursor) after installation.

2. **TeX Live**: https://tug.org/texlive/  
   - Install TeX Live; then add its `bin` directory to your system PATH (e.g. `C:\texlive\2024\bin\windows`).

After installation, check in a **new** terminal:

```powershell
pdflatex --version
```

If that works, build the paper from the **project root**:

```bash
cd paper
pdflatex main
bibtex main
pdflatex main
pdflatex main
```

Or from the project root in one go (PowerShell):

```powershell
cd c:\source\REAEDP\paper
pdflatex main; bibtex main; pdflatex main; pdflatex main
```

The output PDF is `paper/main.pdf`.

## Where to download data

Experiments use **generated data** by default (no download required). To use **your own data**:

### Tabular / CSV

- **UCI ML Repository:** https://archive.ics.uci.edu/ml/index.php — open a dataset, use “Data Folder” to download; put files in **`data/`** (or any path; set paths in `config.json` or script args).
- **Kaggle:** https://www.kaggle.com/datasets — download CSV; put in **`data/`** or set `"input"` in config.
- **Real-data experiment:** The **`csv_entropy_experiment`** task uses a CSV with a numeric column (e.g. `data/y_amazon-google-large.csv`). See **“Using real CSV data in the paper”** below for steps.

### Download data from config (automated)

You can download configured data sources (UCI, Kaggle, or direct URL) into **`data/`** using a single command.

1. **Edit** **`data/download_config.json`**: add or change entries in `"sources"`. Each entry has:
   - **`type`**: `"url"` (direct CSV or zip), `"uci"` (UCI dataset id), `"kaggle"` (dataset: `"owner/dataset-slug"`), or `"kaggle_competition"` (竞赛 slug，如 `house-prices-advanced-regression-techniques`).
   - **`save_as`**: filename under `data/` (e.g. `"y_amazon-google-large.csv"`).
   - For **url**: `"url"`; optionally `"file_in_archive"` if the URL is a zip.
   - For **uci**: `"id"` (e.g. `1276`). Optional: `pip install ucimlrepo`.
   - For **kaggle**: `"id"`; optionally `"file_in_dataset"`. **Kaggle 需登录**：`pip install kaggle`，在 [Kaggle 设置](https://www.kaggle.com/settings) 创建 API Token，将 `kaggle.json` 放到 `~/.kaggle/`（Windows: `C:\Users\<用户名>\.kaggle\`）。
   - For **kaggle_competition**: `"id"` (竞赛 slug)；可选 `"file_in_dataset"`（如 `train.csv`）。需先在 Kaggle 网页上 **接受竞赛规则**（Join Competition / Accept Rules），否则 API 会报错。

2. **Run** from the project root:
   ```bash
   python data/download_data.py
   ```
   Or with a custom config: `python data/download_data.py --config path/to/download_config.json`. List sources only: `python data/download_data.py --list`.

3. **Use** the downloaded CSV in **`config.json`** under **`csv_entropy_experiment`** by setting **`"input"`** to the path (e.g. `"data/y_amazon-google-large.csv"`) and **`"column"`** to the numeric column name.

### Images (image noise test)

- Set **`"input"`** in `config.json` under `"image_noise"` to a file path or folder of images (e.g. `"data/my_images"`). Leave empty to use the built-in demo image.
- **Kaggle / CIFAR / MNIST:** download images, place in a folder, and set that folder as `"input"` in config.

### Summary: where to put downloaded data

| Data type | Put files in | How to use |
|-----------|--------------|------------|
| CSV / tabular | **`data/`** | Set paths in `config.json` or pass `--input` / `--out_csv` when running scripts. |
| Images | Any folder | Set `"image_noise"."input"` in `config.json` or use `--input` with `run_image_noise`. |
| Generated figures | — | Paper figures → **`paper/figs/`**; image noise outputs → **`data/image_noise/`**. |

## Using real CSV data in the paper (e.g. `y_amazon-google-large.csv`)

This type of data (tabular/time-series with a numeric column) is **appropriate** for the paper: it lets you report entropy and differential privacy on a real dataset (e.g. record-linkage or time-series style), and the paper already includes a subsection and figure for it.

### Steps to use your CSV in the paper

1. **Put the CSV in the project**
   - Place the file under **`data/`**, e.g. `data/y_amazon-google-large.csv`.

2. **Install dependency**
   - `pip install pandas` (or `pip install -r requirements.txt`; pandas is listed there).

3. **Configure**
   - In **`config.json`**, set the **`csv_entropy_experiment`** block:
     - **`input`**: path to the CSV, e.g. `"data/y_amazon-google-large.csv"`.
     - **`column`**: name of the numeric column to histogram (e.g. `"y"`).
     - **`bins`**: number of histogram bins (default 30).
     - **`epsilon`**: privacy parameter for Laplace noise (default 1.0).
     - **`max_rows`**: cap rows read (default 100000) to control runtime.
   - Optionally add **`csv_entropy_experiment`** to the **`run`** list so it runs with `python main.py`.

4. **Run the experiment**
   ```bash
   python main.py --run csv_entropy_experiment
   ```
   - This builds a histogram of the chosen column, computes Shannon entropy, adds Laplace noise to the counts, and saves:
   - **`paper/figs/fig_csv_entropy.png`** (for the paper),
     - **`data/csv_entropy_results.csv`** (if `out_csv` is set in config).

5. **Compile the paper**
   - The LaTeX already has a subsection **“Real-data experiment: histogram entropy on CSV data”** and includes **Figure~\ref{fig:csv_entropy}** (`figs/fig_csv_entropy.png`). Run `build.bat` in `paper/`; the new figure will appear in the PDF.

6. **Optional**
   - To try another CSV or column, change **`input`** and **`column`** in config and re-run step 4.

### Where to find more data of this type (更多同类型数据)

The paper uses one real CSV (e.g. `y_amazon-google-large.csv`) with a numeric column. You can use any similar tabular or time-series CSV. Below is a short table of sources where such data can be found; place downloaded files in **`data/`** and set **`input`** and **`column`** in **`config.json`** under **`csv_entropy_experiment`**.

| Source | Type | Notes |
|--------|------|--------|
| **Amazon Product and Google Locations Reviews** (used in paper) | Time-series CSV | [UCI Dataset 1276](https://archive.ics.uci.edu/dataset/1276/amazon+product+and+google+locations+reviews) — file `y_amazon-google-large.csv`; column `y` = # reviews per hour. Cite: Adler et al. (2026), UCI ML Repository, DOI 10.24432/C5SP7K. |
| **UCI ML Repository** | Tabular/CSV | https://archive.ics.uci.edu/ml — choose a dataset with numeric columns; use “Data Folder” to download. |
| **Kaggle** | CSV, time-series | https://www.kaggle.com/datasets — search for “time series”, “retail”, “sensor”, or “record linkage”; pick a numeric column for the histogram. |
| **Amazon-Google / record-linkage** | Product/entity matching | Benchmarks with numeric or categorical columns; extract or use a numeric column (e.g. price, count, id). |
| **TSBench / time-series** | Univariate series | Many time-series CSV files have a single value column `y` or `value`; use that as **`column`** in config. |

After adding a new CSV under **`data/`**, run `python main.py --run csv_entropy_experiment` to regenerate **`paper/figs/fig_csv_entropy.png`** and **`data/csv_entropy_results.csv`**. You can then add a row to the paper’s **Table~\ref{tab:csv_entropy}** (or a “Datasets” table) if you report multiple datasets.
